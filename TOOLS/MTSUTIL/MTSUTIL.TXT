AUTOLOG.INF - ΚΑΤΑΡΓΗΣΗ ΤΗΣ ΚΑΤΑΧΩΡΗΣΗΣ ΜΗΤΡΩΟΥ AUTOLOGON 
=========================================================

Όταν ξεκινάτε τα Windows 98 μπορεί να μην δείτε ένα παράθυρο διαλόγου σύνδεσης
στα Windows ή στο Network.  Μπορεί επίσης να λάβετε ένα από τα ακόλουθα μηνύματα 
σφάλματος:

 - Καμία υπηρεσία δικτύου δεν αποδέχτηκε τη διαδρομή δικτύου που ορίστηκε.

 - Η λειτουργία που ζητήθηκε δεν εκτελέστηκε γιατί ο χρήστης 
   δεν συνδέθηκε στο δίκτυο. Η υπηρεσία που καθορίστηκε δεν υπάρχει.

Επιπλέον, η καρτέλα "Αλλαγή κωδικού πρόσβασης" μπορεί να λείπει από το 
παράθυρο διαλόγου Ιδιότητες: Κωδικοί πρόσβασης.

Αυτή η συμπεριφορά μπορεί να εμφανιστεί εάν υπάρχει η καταχώρηση AutoLogon 
στο ακόλουθο κλειδί μητρώου:

HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Network\Real Mode Net 

      AutoLogon=<x>

   όπου <x> είναι ένας αριθμός.

----------------------------------------------------------
Για να καταργήσετε την καταχώρηση AutoLogon από το μητρώο:
----------------------------------------------------------

1. Επιλέξτε το αρχείο AUTOLOG.INF που βρίσκεται στο φάκελο \Tools\MTSutil
   στο CD των Windows 98.

2. Κάντε κλικ με το δεξιό κουμπί του ποντικιού στο αρχείο AUTOLOG.INF.
	-ή-
   Κρατήστε πατημένο το πλήκτρο SHIFT και πατήστε το πλήκτρο λειτουργικό 
   πλήκτρο F10.

3. Επιλέξτε την εντολή ΕΓΚΑΤΑΣΤΑΣΗ για να καταργήσετε την καταχώρηση AutoLogon.

Η Ανασυγκρότηση Δίσκων για τα Windows 98 (DEFRAG.EXE) μπορεί να ξεκινήσει
και να μην ολοκληρώσει την εργασία της. Εάν συνεχόμενες προσπάθειες για 
την ανασυγκρότηση του δίσκου είναι ανεπιτυχείς, αυτό μπορεί να οφείλεται
σε μια εφαρμογή που γράφει στη μονάδα σκληρού δίσκου. Η Ανασυγκρότηση 
Δίσκων δεν μπορεί να έχει κοινή πρόσβαση στη μονάδα σκληρού δίσκου με άλλες 
εργασίες. 

Μια εφαρμογή που φορτώνεται αυτόματα κατά την εκκίνηση μπορεί να παρεμποδίζει
το DEFRAG. Εκτελέστε το DEFRAG πριν από τη φόρτωση εφαρμογών κατά την εκκίνηση 
προσθέτοντας μια τιμή για το αρχείο DEFRAG.EXE /ALL στο κλειδί RunServicesOnce 
στο μητρώο.

IPAC_OFF.INF - ΑΠΕΝΕΡΓΟΠΟΙΗΣΗ ΑΥΤΟΜΑΤΗΣ ΡΥΘΜΙΣΗΣ ΤΩΝ ΠΑΡΑΜΕΤΡΩΝ IP ΤΟΥ TCP/IP 
=============================================================================

Η στοίβα TCP/IP των Windows 98 υποστηρίζει έναν νέο μηχανισμό για την 
αυτόματη αντιστοίχιση διεύθυνσης για διευθύνσεις IP σε απλές ρυθμίσεις  
τοπικών δικτύων (LAN). Η Αυτόματη διεύθυνση IP είναι μια επέκταση της 
αντιστοίχισης δυναμικών διευθύνσεων IP για προσαρμογείς τοπικού δικτύου. 
Επιτρέπει τη ρύθμιση διευθύνσεων IP χωρίς τη χρήση της αντιστοίχισης στατικών 
διευθύνσεων IP ή την εγκατάσταση ενός διακομιστή Dynamic Host Configuration Protocol
(DHCP). 

Εάν ένας προσαρμογέας τοπικού δικτύου έχει ρυθμιστεί για TCP/IP και έχει ενεργοποιηθεί 
το πλαίσιο επιλογής "Αυτόματη απόδοση διεύθυνσης IP" στην καρτέλα "Διευθύνσεις IP"
στο παράθυρο διαλόγου Ιδιότητες: TCP/IP, τα Windows 98 TCP/IP προσπαθούν να βρουν 
και να χρησιμοποιήσουν μια υπηρεσία DHCP στο συνδεδεμένο δίκτυο για να πάρουν 
μια δυναμικά αντιστοιχισμένη διεύθυνση IP. Εάν δεν βρουν μια υπηρεσία DHCP,
τα Windows 98 χρησιμοποιούν την Αυτόματη διεύθυνση IP αντιστοιχίζοντας στον 
προσαρμογέα μια διεύθυνση IP τάξης Β εκτός του αριθμού δικτύου 169.254.x.x
χώρου διεύθυνσης IP. Με αυτόν τον τρόπο, δύο υπολογιστές μπορούν να συνδεθούν σε 
ένα διανομέα τοπικού δικτύου και να ξεκινήσουν χωρίς τις παραμέτρους μιας 
διεύθυνσης IP και μπορούν να χρησιμοποιούν το δίκτυο TCP/IP για εργασία στο δίκτυο. 
Κάθε υπολογιστής που χρησιμοποιεί την Αυτόματη διεύθυνση IP παίρνει μια διεύθυνση IP 
και ελέγχει για να προσδιορίσει εάν η διεύθυνση IP είναι μοναδική και 
δεν χρησιμοποιείται ήδη στο τοπικό δίκτυο.

Η Αυτόματη διεύθυνση IP ενεργοποιείται από προεπιλογή. Η Αυτόματη διεύθυνση IP
μπορεί να ελεγχθεί εάν οριστεί η τιμή του IPAutoconfigurationEnabled 
(πληκτρολογήστε DWORD) στο κλειδί μητρώου
 
	HKEY_LOCAL_MACHINE\System
	   \CurrentControlSet
	      \Services
	         \VXD
	            \DHCP
	               
Εάν η τιμή του IPAutoconfigurationEnabled δεν είναι μηδενική ή εάν 
λείπει το κλειδί μητρώου, η Αυτόματη διεύθυνση IP είναι ενεργοποιημένη.
Η τιμή 0 στο IPAutoconfigurationEnabled απενεργοποιεί τη δυνατότητα της 
Αυτόματης διεύθυνσης IP. 

Εάν τα Windows 98 δεν εντοπίσουν αρχικά μια υπηρεσία DHCP και ρυθμίσουν 
μια διεύθυνση IP μέσω του IP Autoconfiguration και στη συνέχεια 
εντοπίσουν μια υπηρεσία DHCP στο δίκτυο, τότε τα Windows 98 χρησιμοποιούν 
την υπηρεσία DHCP και αλλάζουν τη διεύθυνση από 169.254.x.x 
σε διευθύνσεις IP που αντιστοιχίζονται από το διακομιστή DHCP.

	-ή-

3. Επιλέξτε την εντολή ΕΓΚΑΤΑΣΤΑΣΗ για να τροποποιήσετε την τιμή του 
   IPAutoconfigurationEnabled.

MAPC_OFF.INF - ΑΠΕΝΕΡΓΟΠΟΙΗΣΗ ΑΝΤΙΣΤΟΙΧΙΣΗΣ ΣΕΛΙΔΩΝ CACHE
=========================================================

Τα Windows 98 υποστηρίζουν την αντιστοίχιση σελίδων cache για
να αυξήσουν το μέγεθος της διαθέσιμης μνήμης για την εκτέλεση εφαρμογών. 
Η αρχιτεκτονική της διαχείρισης μνήμης των Windows 98 διαιρεί τη μνήμη 
σε 2 μέρη:

 - Προστατευμένη λειτουργία cache δίσκου (VCACHE)

 - Μνήμη που έχει εκχωρηθεί για την εκτέλεση προγραμμάτων και την 
   αποθήκευση δεδομένων (VMM) 

Η ανάγνωση από τη μνήμη είναι πολύ ταχύτερη από την ανάγνωση από τον 
σκληρό δίσκο. Η VCACHE βελτιώνει την απόδοση μειώνοντας τις φορές 
που γίνεται πρόσβαση στον σκληρό δίσκο.  Μια διαδικασία στη μνήμη 
απαιτεί ένα συγκεκριμένο κομμάτι κώδικα. Εάν αυτό το κομμάτι 
βρίσκεται στην VCACHE, η χρήση του και η πρόσβαση σε αυτό είναι πολύ 
ταχύτερη παρά εάν έπρεπε να αναγνωστεί από ένα αρχείο στο δίσκο. 
Για παράδειγμα, εάν έπρεπε να ξεκινήσετε μια εφαρμογή, να την κλείσετε και 
μετά από λίγο να την ξεκινήσετε ξανά, ο χρόνος εκκίνησης της εφαρμογής 
θα είχε μειωθεί σημαντικά. Αυτό είναι αποτέλεσμα της αντιγραφής μεγάλου 
μέρους της εφαρμογής από την VCACHE αντί για τον σκληρό δίσκο.

Η μνήμη που εκχωρείται για το cache του δίσκου δεν είναι διαθέσιμη 
για την εκτέλεση διαδικασιών. Η μείωση του ποσού της διαθέσιμης μνήμης 
για την εκτέλεση εφαρμογών έχει αρνητική επίδραση στην απόδοση επειδή 
αυξάνει το μέγεθος των δεδομένων που μετατίθενται από τη μνήμη στο 
σκληρό δίσκο. Για άλλη μια φορά, η ανάγνωση από τη μνήμη είναι πολύ 
ταχύτερη από το αρχείο μετάθεσης στο σκληρό δίσκο. 

Στα Windows 95, υπάρχουν δύο εμφανίσεις κάποιου κώδικα στη φυσική μνήμη. 
Μία εμφάνιση είναι αποθηκευμένη στο cache του δίσκου για να μειώνει το 
χρόνο που απαιτείται για τη φόρτωση της εφαρμογής εάν αυτή ξαναζητηθεί 
και μια δεύτερη στη VMM για την εφαρμογή που εκτελείται. Εάν εξαλειφθεί 
η παρουσία από τα δεδομένα στο cache του δίσκου θα αυξήσει το ποσό της μνήμης 
που θα είναι διαθέσιμη στη VMM, αλλά θα είχε αρνητικό αντίκτυπο στο χρόνο 
επαναφόρτωσης της εφαρμογής.

Τα Windows 98 υποστηρίζουν αντιστοίχιση σελίδων cache για να αυξήσουν 
το μέγεθος της διαθέσιμης μνήμης για την εκτέλεση εφαρμογών. Η αντιστοίχιση 
I/O στη μνήμη από το cache εξαλείφει την άσκοπη διπλή αντιγραφή δεδομένων κατά 
την εκτέλεση της εφαρμογής. Ο κωδικός της εφαρμογής φορτώνεται στη VCACHE 
και μετά εκτελείται από το cache αντί να αντιγραφεί σε κάποιο άλλο μέρος της 
μνήμης πριν εκτελεστεί. Όταν η εφαρμογή κλείσει, η διαχείριση μνήμης 
προσδιορίζει το χώρο που χρησιμοποιούσε η εφαρμογή ως χώρο που δεν χρησιμοποιείται 
πλέον, για να διατηρήσει τα πλεονεκτήματα της προσωρινής αποθήκευσης--εάν η εφαρμογή 
δεν ξεκινήσει ξανά, αυτός ο χώρος δίνεται σε άλλες εργασίες. Εάν η εφαρμογή 
ξεκινήσει ξανά, υπάρχει ακόμη στο χώρο προσωρινής αποθήκευσης, διατηρώντας 
βελτιωμένο χρόνο επαναφόρτωσης της εφαρμογής.

Τα Windows 98 αντιστοιχίζουν σελίδες cache από προεπιλογή. Αυτή η 
βελτιστοποίηση της απόδοσης μπορεί να απενεργοποιηθεί με τη δημιουργία 
μιας καταχώρησης MapCache στο κλειδί μητρώου
 
	HKEY_LOCAL_MACHINE\System
	   \CurrentControlSet
	      \Services
	         \VXD
	            \VMM

Εάν προσθέσετε την τιμή "NameNumericTail=0" στο μητρώο (όπως

Προσθαφαίρεση προγραμμάτων από τον Πίνακα Ελέγχου, ορισμένα προγράμματα

Όταν προσθέτετε την τιμή "NameNumericTail=0" στο μητρώο, η Εγκατάσταση

περιγράφονται σε αυτό το άρθρο μπορεί να συμβούν. Αυτό το θέμα μπορεί 
να επηρεάσει οποιοδήποτε πρόγραμμα έχει εγκατασταθεί στο φάκελο Program 
Files.

μητρώου λειτουργεί όπως αναμένεται, ορισμένα στοιχεία των Windows 98 μπορεί 
να μην λειτουργούν κανονικά.

1. Διαγράψτε την τιμή "NameNumericTail=0" από το ακόλουθο κλειδί μητρώου:

      Hkey_Local_Machine\System\CurrentControlSet\Control\FileSystem

2. Χρησιμοποιώντας την Εξερεύνηση των Windows ή το εικονίδιο Ο 
   Υπολογιστής μου, μετονομάστε το φάκελο Program Files σε Programs.fld.

3. Μετονομάστε το φάκελο Programs.fld σε Program Files.

   Σημείωση: Η μετονομασία του φακέλου έχει ως αποτέλεσμα τη δημιουργία ενός νέου 
         σύντομου ονόματος για το φάκελο.

NOIDE.INF - ΚΑΤΑΡΓΗΣΗ ΤΗΣ ΚΑΤΑΧΩΡΗΣΗΣ ΜΗΤΡΩΟΥ NOIDE
===================================================

Εάν ο υπολογιστής σας περιέχει έναν ελεγκτή σκληρού δίσκου PCI-IDE 
που χρησιμοποιεί σειριακή σύνδεση ανάμεσα στα δύο κανάλια IDE, μπορεί
να προκύψουν τα παρακάτω θέματα:

 - Η πρόσβαση στο σύστημα αρχείων 32-bit και η εικονική μνήμη 32-bit να μην 
   είναι διαθέσιμες.

 - Η Διαχείριση συσκευών να εμφανίζει ένα θαυμαστικό σε έναν κίτρινο κύκλο 
   στο πρωταρχικό και το δευτερεύον κανάλι IDE.

Η κατάργηση και η επανεγκατάσταση του ελεγκτή σκληρού δίσκου δεν επιλύει το 
ζήτημα.

Αυτό μπορεί να συμβεί εάν το πρόγραμμα οδήγησης σε προστατευμένη κατάσταση λειτουργίας
για τον ελεγκτή σκληρού δίσκου δεν προετοιμάστηκε σωστά κατά την προηγούμενη έναρξη
των Windows 98. Όταν συμβεί αυτό, μια καταχώρηση NOIDE τοποθετείται στο μητρώο, 
αποτρέποντας μελλοντικές προσπάθειες των Windows 98 για προετοιμασία του προγράμματος 
οδήγησης σε προστατευμένη κατάσταση λειτουργίας.

Αυτό το θέμα μπορεί να παρουσιαστεί με έναν ελεγκτή IDE που απαιτεί 
σειριακή σύνδεση ανάμεσα στο πρωταρχικό και το δευτερεύων κανάλι IDE. 
Τα προγράμματα οδήγησης σε προστατευμένη κατάσταση λειτουργίας για αυτούς τους 
ελεγκτές IDE μπορεί να αποτύχουν να προετοιμαστούν εάν συμβεί μία από τις 
παρακάτω καταστάσεις:

 - Ένα κανάλι IDE έχει έναν υποστηριζόμενο σκληρό δίσκο και το δεύτερο κανάλι
   έχει ένα CD-ROM ή άλλο τύπο μονάδας δίσκου που απαιτεί την φόρτωση προγραμμάτων
   οδήγησης σε πραγματική κατάσταση λειτουργίας. Λόγω της σειριακής σύνδεσης ανάμεσα 
   στα δύο κανάλια IDE, είναι αδύνατη η πρόσβαση στο σκληρό δίσκο σε προστατευμένη 
   κατάσταση λειτουργίας και η χρήση της άλλης συσκευής σε πραγματική κατάσταση λειτουργίας. 
   Αυτό έχει ως αποτέλεσμα την αποτυχία της προετοιμασίας του προγράμματος 
   οδήγησης σε προστατευμένη κατάσταση και τοποθετείται στο μητρώο η καταχώρηση 
   NOIDE για να αποτρέψει μελλοντικά σφάλματα. Και οι δύο συσκευές δίσκου λειτουργούν 
   πλέον σε πραγματική κατάσταση λειτουργίας.

 - Το πρόγραμμα οδήγησης του ελεγκτή IDE έχει καταργηθεί και επανεγκατασταθεί με 
   μη αυτόματο τρόπο από τη Διαχείριση συσκευών ή το πρόγραμμα οδήγησης σε προστατευμένη 
   κατάσταση λειτουργίας έχει απενεργοποιηθεί και στη συνέχεια έχει ενεργοποιηθεί ξανά. 
   Ορισμένα προγράμματα οδήγησης ελεγκτών PCI δεν έχουν σχεδιαστεί για δυναμική 
   ενεργοποίηση και απενεργοποίηση, γεγονός που μπορεί να προκαλέσει την αποτυχία 
   προετοιμασίας του προγράμματος οδήγησης σε προστατευμένη κατάσταση λειτουργίας.

Για να προκαλέσετε μια νέα προσπάθεια των Windows 98 για την προετοιμασία του 
προγράμματος οδήγησης σε προστατευμένη κατάσταση λειτουργίας IDE, καταργήστε 
την καταχώρηση NOIDE από το παρακάτω κλειδί μητρώου:

   HKEY_LOCAL_MACHINE\SYSTEM\CURRENTCONTROLSET\SERVICES\VXD\IOS

------------------------------------------------------

------------------------------------------------------

1. Επιλέξτε το αρχείο NOIDE.INF που βρίσκεται στο φάκελο \Tools\MTSutil 
   στο CD των Windows 98.

2. Κάντε κλικ με το δεξιό κουμπί του ποντικιού στο αρχείο NOIDE.INF.
	-ή-
   Κρατήστε πατημένο το πλήκτρο SHIFT και πατήστε το πλήκτρο λειτουργίας F10.

3. Επιλέξτε την εντολή ΕΓΚΑΤΑΣΤΑΣΗ για να καταργήσετε την καταχώρηση NOIDE.

Μετά την ενημέρωση του μητρώου, επανεκκινήστε τα Windows 98. Τα Windows 98 
θα προσπαθήσουν να προετοιμάσουν το πρόγραμμα οδήγησης σε πραγματική κατάσταση 
λειτουργίας του ελεγκτή. Εάν όλα λειτουργήσουν κανονικά, το σύστημα αρχείων 
και η εικονική μνήμη θα λειτουργούν σε κατάσταση λειτουργίας 32-bit και η Διαχείριση 
συσκευών δεν θα εμφανίζει ένα θαυμαστικό σε έναν κίτρινο κύκλο για τα κανάλια IDE.

Εάν δεν έγινε σωστή προετοιμασία του προγράμματος οδήγησης σε προστατευμένη κατάσταση 
λειτουργίας, θα εμφανιστεί ένα μήνυμα σφάλματος και η καταχώρηση μητρώου NOIDE 
θα ξαναδημιουργηθεί. Τα Windows 98 θα χρησιμοποιήσουν το σύστημα αρχείων σε συμβατή 
λειτουργία MS-DOS την επόμενη φορά που θα ξεκινήσετε τον υπολογιστή σας.

Μετά την αναβάθμιση σε Windows 98, μπορεί να εμφανιστεί το ακόλουθο 
μήνυμα όταν κάνετε διπλό κλικ σε ένα εικονίδιο στην επιφάνεια εργασίας:

   Εντοπίστηκε το Microsoft Network

   Εντοπίστηκε ότι το MSN έχει ήδη εγκατασταθεί σε αυτόν τον υπολογιστή.

   Εάν εγγραφείτε τώρα στο MSN (ή εάν θέλετε να εγγραφείτε), πρέπει να κάνετε 
   κλικ στο κουμπί "Εγκατάσταση του MSN" παρακάτω για να συνδεθείτε στο MSN 
   μέσω του Internet Explorer.

   Για να συνεχίσετε να χρησιμοποιείτε τον Internet Explorer για διαφορετικούς 
   σκοπούς εκτός από την πρόσβαση στο MSN, κάντε κλικ στο κουμπί "Internet Explorer" 
   παρακάτω.

   [ ] Να μην εμφανιστεί ξανά αυτό το παράθυρο διαλόγου.

   [’κυρο] [Internet Explorer] [Εγκατάσταση του MSN]

Τα Windows 98 προσθέτουν αυτό το μήνυμα ως μια υπηρεσία στους πελάτες του MSN. 
Αυτό το μήνυμα προστίθεται στα εικονίδια της επιφάνειας εργασίας εάν ανιχνευθεί μια 
παλιότερη έκδοση του προγράμματος-πελάτη του The Microsoft Network (MSN) 
ή εάν δεν ολοκληρωθεί το δεύτερο στάδιο της διαδικασίας εγκατάστασης του MSN. 
Τα Windows 98 δεν ενημερώνουν αυτόματα το πρόγραμμα-πελάτη του MSN. 

Για να απενεργοποιήσετε αυτήν την ειδοποίηση, αναβαθμίστε το πρόγραμμα-πελάτη 
του MSN κάνοντας κλικ στο κουμπί "Εγκατάσταση του MSN" ή όταν εμφανιστεί το 
μήνυμα, κάντε κλικ στο πλαίσιο ελέγχου "Να μην εμφανιστεί ξανά αυτό το παράθυρο διαλόγου." 

Εάν το εικονίδιο του Outlook 98 της επιφάνειας εργασίας εξακολουθεί να σας ζητάει 
να ενημερώσετε το πρόγραμμα-πλάτη του MSN, εκτελέστε το αρχείο CHNGINBX.EXE 
για να απενεργοποιήσετε το μήνυμα.
